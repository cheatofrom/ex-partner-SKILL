# Relationship History 分析 Prompt

## 任务

你将收到 **{name}** 的原材料（聊天记录、备忘录、分手信等）。
从中提取他/她的恋爱历史与核心事件，用于构建 Relationship History。

**原则：只提取与感情进展、矛盾、约定相关的内容。不要推断，有依据才写，没有就标注"原材料不足"。**

---

## 提取维度

### 1. 恋爱里程碑与时间线

从原材料中识别：
- 认识的契机与地点
- 确认关系的节点
- 重要的纪念日、生日怎么过的
- 感情升温/降温的转折点
- 分手的导火索与真实原因

```
输出格式：
相识：[描述]
确认关系：[时间/事件]
感情转折：[列表]
分手原因：[描述]
```

### 2. 核心矛盾与争吵记录

从吵架记录中提取：
- 最频繁吵架的话题（如：回消息慢、异地、异性朋友、消费观）
- 典型的争吵循环（谁先发火 -> 谁冷暴力 -> 谁先低头）
- 遗留的未解决问题（每次吵架都会翻出来的旧账）

```
输出格式：
核心矛盾：[列表]
争吵模式：[步骤描述]
历史旧账：[列表]
```

### 3. 约定与画过的大饼

- 承诺过但没做到的事（如：说好去旅游、说好见父母）
- 共同规划过的未来（买房、养宠物、结婚计划）
- 专属的约定（如：每天必须说早安）

```
输出格式：
未兑现承诺：[列表]
未来规划：[列表]
专属约定：[列表]
```

### 4. 专属回忆与梗

- 只有你们懂的内部笑话/称呼
- 经常去的地方（某家餐厅、某个城市）
- 送过的印象深刻的礼物

```
输出格式：
专属称呼/梗：[列表]
重要地点：[列表]
重要礼物：[列表]
```

### 5. 原材料金句库

他/她明确表达过的感情观、伤人的话、或者感人的承诺（直接引用原话）：

```
- "[原话或总结]"
- "[原话或总结]"
```
